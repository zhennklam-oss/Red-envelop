# 内网穿透配置说明 - 让外网朋友也能访问

## 🌐 什么是内网穿透？

内网穿透可以让您的本地服务器通过公网访问，这样不同局域网的朋友也能打开您的红包链接。

---

## 📋 推荐方案对比

| 工具 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **ngrok** | 稳定、速度快、功能强大 | 免费版有连接数限制 | ⭐⭐⭐⭐⭐ |
| **cpolar** | 国内服务、中文界面、稳定 | 免费版有流量限制 | ⭐⭐⭐⭐⭐ |
| **localtunnel** | 完全免费、无需注册 | 速度较慢、不太稳定 | ⭐⭐⭐ |
| **frp** | 完全免费、可自建 | 需要有公网服务器 | ⭐⭐⭐⭐ |

---

## 🚀 方案一：使用 ngrok（推荐国外用户）

### 1. 安装 ngrok

访问 [ngrok.com](https://ngrok.com/) 注册账号并下载。

Windows 用户可以直接下载 exe 文件。

### 2. 配置 ngrok

```bash
# 登录（使用您的 authtoken）
ngrok config add-authtoken YOUR_AUTH_TOKEN
```

### 3. 启动内网穿透

```bash
# 在一个终端窗口运行
ngrok http 3000
```

### 4. 获取公网地址

ngrok 启动后会显示类似这样的信息：

```
Forwarding  https://abc123.ngrok.io -> http://localhost:3000
```

### 5. 配置环境变量

创建 `.env` 文件（复制 `.env.example`）：

```bash
PUBLIC_URL="https://abc123.ngrok.io"
```

### 6. 启动应用

```bash
npm run dev
```

## 🚀 方案二：使用 cpolar（推荐国内用户）

### 1. 安装 cpolar

访问 [cpolar.com](https://www.cpolar.com/) 注册账号并下载客户端。

### 2. 登录 cpolar

```bash
cpolar authtoken YOUR_AUTH_TOKEN
```

### 3. 启动内网穿透

```bash
cpolar http 3000
```

或者使用图形界面：
- 打开 cpolar 客户端
- 选择"隧道管理" → "创建隧道"
- 本地地址：`3000`
- 域名类型：随机域名（免费）或自定义域名（付费）

### 4. 获取公网地址

cpolar 会分配一个公网地址，例如：

```
http://abc123.cpolar.cn
```

### 5. 配置环境变量

创建 `.env` 文件：

```bash
PUBLIC_URL="http://abc123.cpolar.cn"
```

### 6. 启动应用

```bash
npm run dev
```

## 🚀 方案三：使用 localtunnel（最简单）

### 1. 安装 localtunnel

```bash
npm install -g localtunnel
```

### 2. 启动应用

```bash
npm run dev
```

### 3. 在另一个终端启动 localtunnel

```bash
lt --port 3000
```

### 4. 获取公网地址

localtunnel 会显示：

```
your url is: https://abc-123.loca.lt
```

### 5. 配置环境变量

创建 `.env` 文件：

```bash
PUBLIC_URL="https://abc-123.loca.lt"
```

### 6. 重启应用

```bash
# 停止当前服务（Ctrl+C），然后重新启动
npm run dev
```

## 🚀 方案四：使用 frp（高级用户）

### 前提条件

您需要有一台拥有公网IP的服务器（如阿里云、腾讯云等）。

### 1. 在服务器上配置 frps（服务端）

下载 frp：[https://github.com/fatedier/frp/releases](https://github.com/fatedier/frp/releases)

创建 `frps.ini`：

```ini
[common]
bind_port = 7000
```

启动服务端：

```bash
./frps -c frps.ini
```

### 2. 在本地配置 frpc（客户端）

创建 `frpc.ini`：

```ini
[common]
server_addr = YOUR_SERVER_IP
server_port = 7000

[red-envelope]
type = http
local_port = 3000
custom_domains = your-domain.com
```

启动客户端：

```bash
./frpc -c frpc.ini
```

### 3. 配置环境变量

```bash
PUBLIC_URL="http://your-domain.com"
```

## 📝 完整使用流程

### 步骤 1：启动内网穿透服务

选择上述任一方案，启动内网穿透服务并获取公网地址。

### 步骤 2：配置环境变量

在项目根目录创建 `.env` 文件：

```bash
PUBLIC_URL="https://your-public-url.com"
```

### 步骤 3：启动应用

```bash
npm run dev
```

## ⚠️ 注意事项

### 安全性

1. **管理员密钥保护**
   - 不要将创建者链接（带 `?key=creator_secret_123`）公开分享
   - 只分享普通链接给访客

2. **HTTPS 推荐**
   - 尽量使用 HTTPS 的内网穿透服务
   - ngrok 和 cpolar 免费版都支持 HTTPS

3. **访问控制**
   - 内网穿透会让您的服务完全暴露在公网
   - 使用完毕后记得关闭内网穿透服务

### 性能

1. **速度**
   - 内网穿透会增加一定延迟
   - 国内用户建议使用 cpolar
   - 国外用户建议使用 ngrok

2. **流量限制**
   - 免费版通常有流量或连接数限制
   - 照片不要太大（建议 < 2MB）

3. **稳定性**
   - 免费服务可能不太稳定
   - 重要场合建议使用付费版

### 域名变化

某些免费服务（如 localtunnel）每次启动会分配新的域名：

1. 每次启动后更新 `.env` 文件中的 `PUBLIC_URL`
2. 重启应用服务器
3. 重新复制分享链接

## 🔧 常见问题

### Q: 内网穿透服务断开了怎么办？

**A:** 重新启动内网穿透服务，如果域名变了，需要更新 `.env` 文件并重启应用。

### Q: 朋友访问很慢怎么办？

**A:**
1. 尝试更换内网穿透服务
2. 压缩照片大小
3. 使用付费版内网穿透服务

### Q: 可以使用自己的域名吗？

**A:** 可以，但通常需要：
1. 使用付费版内网穿透服务
2. 或者使用 frp + 自己的服务器

### Q: 安全吗？

**A:**
- 内网穿透本身是安全的
- 但要注意不要泄露创建者链接
- 使用完毕后关闭服务
